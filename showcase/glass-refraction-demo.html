The math

Surface equations

Ported from @hashintel/refractive (MIT/Apache-2.0). These functions describe how a curved glass surface displaces light based on position — highest at the bezel rim, flat at the center.

convexCircle(x)
convex(x)
concave(x)
lip(x)

Snell's law says light bends when it crosses materials of different refractive index. At a glass edge, the surface is steeply curved — so light bends a lot. In the flat interior, it barely bends.

The lip(x) function (blue, thicker) describes this shape exactly — high near the edge (x ≈ 0), falling quickly as you move toward the center (x → 1). We use this as the displacement magnitude for each pixel in our feDisplacementMap.

The three helper curves (convexCircle, convex, concave) are blended with a smootherstep function — a 5th-degree polynomial that eliminates derivative discontinuities at the seam, giving the rim its soft, physically plausible curvature.

In the 64×64 displacement map we generate at runtime, each pixel's R channel encodes the X offset and G channel encodes the Y offset, with 128 as neutral (no shift). Edge pixels get values far from 128; center pixels stay close to it.

// Surface equations — ported from @hashintel/refractive (MIT/Apache-2.0)
// These model the displacement magnitude at each point on a glass surface.
//
//   convexCircle : quarter-circle arc (outer curvature)
//   convex       : softer polynomial convex curve
//   concave      : inverse of convexCircle (inner curvature)
//   lip          : the final blended curve — high at the rim, low at center
//                  smootherstep controls the blend between convex and concave

function smootherstep(x) {
  x = Math.min(1, Math.max(0, x));
  return x*x*x * (x * (x*6 - 15) + 10);   // 5th-degree, C²-smooth at 0 and 1
}

function convexCircle(x) {
  return Math.sqrt(1 - (1-x)*(1-x));           // quarter-circle arc
}

function convex(x) {
  return Math.pow(1 - Math.pow(1-x, 4), 0.25); // softer polynomial
}

function concave(x) {
  return 1 - convexCircle(x);                   // inverse of convexCircle
}

function lip(x) {
  const t   = smootherstep(x);
  const cvx = convex(Math.min(1, x*2));         // convex side of rim
  const ccv = concave(x) + 0.1;                // concave side (+bias)
  return cvx * (1-t) + ccv * t;               // smoothstep blend
}
// Build a SIZE×SIZE displacement map ImageData and return it as a PNG data URL.
//
// For each pixel we compute:
//   edgeDist = min distance to any edge, normalised 0 (edge) → 1 (center)
//   mag      = lip(1 - edgeDist)   ← high near edge, near-zero at center
//   (vx, vy) = unit vector pointing outward from center
//
// R channel encodes X offset:  128 + vx * mag * 127
// G channel encodes Y offset:  128 + vy * mag * 127
// 128 in both = neutral (feDisplacementMap does nothing at that pixel).

function buildDisplacementMap(size) {
  const canvas  = document.createElement('canvas');
  canvas.width   = canvas.height = size;
  const ctx     = canvas.getContext('2d');
  const img     = ctx.createImageData(size, size);

  for (let py = 0; py < size; py++) {
    for (let px = 0; px < size; px++) {
      const nx       = px / (size-1);            // 0..1 left→right
      const ny       = py / (size-1);            // 0..1 top→bottom
      const edgeDist = Math.min(nx, 1-nx, ny, 1-ny) * 2;
      const mag      = lip(1 - edgeDist);         // high at edge
      const vx       = nx - 0.5;
      const vy       = ny - 0.5;
      const len      = Math.sqrt(vx*vx + vy*vy) || 1;
      const i        = (py*size + px) * 4;

      img.data[i]   = Math.round(128 + (vx/len) * mag * 127); // R → X
      img.data[i+1] = Math.round(128 + (vy/len) * mag * 127); // G → Y
      img.data[i+2] = 0;
      img.data[i+3] = 255;
    }
  }
  ctx.putImageData(img, 0, 0);
  return canvas.toDataURL('image/png');
}
